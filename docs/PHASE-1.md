# 第一阶段目标

第一阶段不是完成最终一键部署。

第一阶段只做：

1. 建立独立交付仓库
2. 建立统一顶层编排
3. 建立统一环境变量规范
4. 建立基础 CI 校验

## 第一阶段完成标准

- 仓库独立存在
- 有顶层 `docker-compose.yml`
- 有 `.env.example`
- 有 Nginx 模板
- 有基础脚本
- 有 GitHub Actions 校验

## 第一阶段明确不做

- 生产迁移
- WordPress 初始化自动化
- 插件打包自动化
- ACF 自动导出
- TLS 自动签发
- 正式环境无人值守发布

## 为什么这样做

因为现在最重要的是先把“系统边界”固定下来。

边界没固定前，做安装器、自动发布、正式迁移都太早。
